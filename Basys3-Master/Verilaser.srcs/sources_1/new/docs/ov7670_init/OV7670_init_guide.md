# OV7670 카메라 초기화 가이드

## 전체 그림

OV7670은 카메라 센서 칩이다. 
전원을 켜면 기본 설정(VGA 640x480, YUV 컬러)으로 동작하는데, 우리가 원하는 설정(QVGA 320x240, RGB565 컬러)으로 바꿔줘야 한다.

설정을 바꾸려면 카메라 내부의 **레지스터**에 값을 써야 한다. 
레지스터란 카메라 안에 있는 작은 설정 저장소로, 각각 고유한 주소가 있고, 거기에 값을 쓰면 카메라 동작이 바뀐다.

```
[FPGA] ---SCCB 통신--→ [OV7670 레지스터] → 카메라 동작 변경
```

## SCCB 통신이란

레지스터에 값을 쓰려면 **SCCB**(Serial Camera Control Bus)라는 통신 방식을 사용한다. I2C와 거의 같은 2선 통신이다.

- **SCL** (Serial Clock): FPGA가 만드는 클록 신호
- **SDA** (Serial Data): 데이터를 주고받는 신호

### 한 번의 쓰기 과정 (3-Phase Write)

카메라 레지스터 하나를 설정하려면, 아래 순서로 3바이트를 보낸다:

```
[START] → [카메라 주소: 0x42] → [레지스터 주소] → [설정 값] → [STOP]
```

예를 들어 COM7 레지스터(주소 0x12)에 0x80(리셋)을 쓰려면:

```
[START] → [0x42] → [0x12] → [0x80] → [STOP]
         "OV7670아"  "COM7에"   "0x80 써"
```

### 비트 전송 방법

각 바이트(8비트)는 MSB(최상위 비트)부터 한 비트씩 보낸다:

```
        비트7  비트6  비트5  비트4  비트3  비트2  비트1  비트0   ACK
SCL:  __/‾‾\__/‾‾\__/‾‾\__/‾‾\__/‾‾\__/‾‾\__/‾‾\__/‾‾\__/‾‾\__
SDA:  ==D7====D6====D5====D4====D3====D2====D1====D0====X======

      ↑ SCL이 HIGH일 때 SDA 값을 읽는다
```

- SCL이 LOW일 때: SDA 값을 바꿔도 됨 (준비 단계)
- SCL이 HIGH일 때: SDA 값이 확정됨 (카메라가 이 순간 값을 읽음)
- ACK: 카메라가 "잘 받았다"고 응답하는 구간

### START / STOP 조건

```
START: SCL이 HIGH인 상태에서 SDA가 HIGH→LOW
STOP:  SCL이 HIGH인 상태에서 SDA가 LOW→HIGH
```

이 두 신호로 통신의 시작과 끝을 알린다.

## 초기화 순서

### 1단계: 소프트웨어 리셋

```
COM7(0x12) = 0x80
```

카메라 내부 설정을 전부 공장 초기값으로 되돌린다. 리셋 후 **30ms 대기**해야 카메라가 안정된다.

### 2단계: 기본 튜닝값 설정

감마 보정, 자동 노출(AEC), 자동 게인(AGC), 화이트밸런스(AWB) 등 화질 관련 설정을 잡는다. 이 값들은 OV7670 제조사 권장값이다.

### 3단계: 해상도 설정 (QVGA)

```
COM7(0x12)  = 0x11   → QVGA 모드 활성화
COM3(0x0C)  = 0x04   → 다운샘플링 활성화
COM14(0x3E) = 0x19   → 픽셀 클록 분주
0x70~0x73, 0xA2      → 스케일링 세부 설정
```

VGA(640x480) 원본 영상을 절반으로 줄여서 QVGA(320x240)로 만든다.

### 4단계: 이미지 윈도우 설정

```
HSTART(0x17) = 0x15   → 수평 시작 위치
HSTOP(0x18)  = 0x03   → 수평 끝 위치
VSTART(0x19) = 0x03   → 수직 시작 위치
VSTOP(0x1A)  = 0x7B   → 수직 끝 위치
```

센서의 어느 영역을 잘라서 출력할지 정한다. 사진을 찍을 때 프레임을 잡는 것과 같다.

### 5단계: 컬러 포맷 설정 (RGB565)

```
COM7(0x12)  = 0x14   → QVGA + RGB 모드
COM15(0x40) = 0xD0   → RGB565 포맷
```

카메라 출력 색상 형식을 RGB565로 설정한다. RGB565란 한 픽셀을 16비트(2바이트)로 표현하는 방식이다:

```
[RRRRR GGGGGG BBBBB]
 빨강5  초록6   파랑5  = 총 16비트
```

**주의**: 이 단계에서 COM7에 0x04가 아니라 **0x14**를 써야 한다. 0x04는 RGB만 설정하고 QVGA 비트를 날려서, 3단계에서 설정한 QVGA가 풀려버린다.

### 6단계: 컬러 매트릭스

센서 원시 데이터를 사람 눈에 자연스러운 색으로 변환하는 계수를 설정한다.

## FPGA 구현 구조

```
OV7670_Init_Controller
├── sccb_master       : SCCB 프로토콜 처리 (START/STOP/WRITE/ACK)
├── sccb_sequencer    : 레지스터를 순서대로 하나씩 쓰는 상태 머신
└── OV7670_reg_rom    : 써야 할 레지스터 목록 저장 (.mem 파일에서 로드)
```

### 동작 흐름

```
리셋 해제
    │
    ▼
[레지스터 0번 쓰기]
    │  START → 0x42 → reg_addr → reg_data → STOP
    ▼
[딜레이 대기] ─── 0번(SW Reset)이면 30ms, 나머지는 1ms
    │
    ▼
[레지스터 1번 쓰기]
    │
    ▼
   ...반복...
    │
    ▼
[레지스터 65번 쓰기]
    │
    ▼
[DONE] ── 초기화 완료, 카메라가 설정대로 영상 출력 시작
```

## 카메라 출력 신호

초기화가 끝나면 카메라는 아래 신호를 통해 영상 데이터를 보낸다:

```
VSYNC:  ‾‾\____/‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾\____/‾‾
         프레임    한 프레임 (240 라인)           프레임
         시작                                    시작

HREF:   ____/‾‾‾‾‾‾‾\___/‾‾‾‾‾‾‾\___    ___/‾‾‾‾‾‾‾\____
             라인 0       라인 1    ...      라인 239

PCLK:   _/‾\_/‾\_/‾\_/‾\_/‾\_/‾\_ ...
         ↑   ↑   ↑
         바이트 바이트 바이트 ...

DATA:   [B0][B1][B2][B3][B4][B5] ...
         \_픽셀0_/ \_픽셀1_/ \_픽셀2_/
```

- **VSYNC**: 새 프레임 시작을 알림
- **HREF**: HIGH 구간에서 유효한 픽셀 데이터가 나옴
- **PCLK**: 매 클록마다 1바이트 출력
- **DATA[7:0]**: 8비트 데이터. 2바이트가 모여 1픽셀(RGB565)

## OV7670 주요 레지스터 맵

OV7670에는 약 200개의 레지스터가 있다. 이 중 초기화에 사용하는 주요 레지스터를 기능별로 정리하면:

### 핵심 제어 레지스터

| 주소 | 이름 | 역할 |
|------|------|------|
| 0x12 | COM7 | **가장 중요한 레지스터.** 해상도(VGA/QVGA), 출력 포맷(YUV/RGB), 소프트웨어 리셋을 제어 |
| 0x13 | COM8 | 자동 노출(AEC), 자동 게인(AGC), 화이트밸런스(AWB) ON/OFF |
| 0x0C | COM3 | 다운샘플링(DCW) 활성화. QVGA 사용 시 필수 |
| 0x3E | COM14 | 픽셀 클록(PCLK) 분주 설정. 해상도 줄이면 클록도 줄여야 함 |
| 0x40 | COM15 | RGB565/RGB555 선택, 출력 데이터 범위 설정 |
| 0x42 | COM17 | 컬러바 테스트 패턴 ON/OFF |
| 0x3A | TSLB | 출력 바이트 순서 설정. OV 공식 문서에서 "very important" 표기 |

### COM7 (0x12) 비트 상세

이 레지스터 하나에 여러 설정이 들어있어서, 값을 쓸 때 주의가 필요하다:

```
bit[7]: 1 = 소프트웨어 리셋 (자동으로 0으로 돌아감)
bit[6]: 예약
bit[5]: CIF 포맷
bit[4]: QVGA 선택     ← 0이면 VGA, 1이면 QVGA
bit[3]: QCIF 포맷
bit[2]: RGB 선택       ← 0이면 YUV, 1이면 RGB
bit[1]: 컬러바
bit[0]: Raw RGB
```

QVGA + RGB를 쓰려면 bit[4]과 bit[2]를 모두 1로 해야 하므로 **0x14** (0001_0100)가 된다.

> **함정**: 해상도 설정(3단계)에서 COM7 = 0x11을 쓰고, 컬러 설정(5단계)에서 COM7 = 0x04를 쓰면
> 두 번째 쓰기가 첫 번째를 덮어써서 QVGA 비트가 사라진다.
> 반드시 마지막에 쓰는 값에 **모든 필요한 비트를 합쳐서** 써야 한다. → `0x14`

### 스케일링 레지스터

| 주소 | 이름 | 역할 |
|------|------|------|
| 0x70 | SCALING_XSC | 수평 스케일 팩터 |
| 0x71 | SCALING_YSC | 수직 스케일 팩터 |
| 0x72 | SCALING_DCWCTR | 다운샘플링 제어 |
| 0x73 | SCALING_PCLK_DIV | PCLK 분주 비율 |
| 0xA2 | SCALING_PCLK_DELAY | PCLK 딜레이 |

VGA → QVGA 변환 시 내부적으로 2x2 픽셀을 1픽셀로 합치는데, 이 과정을 제어한다.

### 이미지 윈도우 레지스터

| 주소 | 이름 | 역할 |
|------|------|------|
| 0x17 | HSTART | 수평 캡처 시작 위치 (상위 비트) |
| 0x18 | HSTOP | 수평 캡처 종료 위치 (상위 비트) |
| 0x32 | HREF | HSTART/HSTOP 하위 비트 + HREF 제어 |
| 0x19 | VSTART | 수직 캡처 시작 위치 (상위 비트) |
| 0x1A | VSTOP | 수직 캡처 종료 위치 (상위 비트) |
| 0x03 | VREF | VSTART/VSTOP 하위 비트 |

센서 전체 영역 중 어디서부터 어디까지를 출력할지 결정한다. 이 값이 잘못되면 이미지가 밀리거나 잘린다.

### 화질 튜닝 레지스터

| 주소 범위 | 이름 | 역할 |
|-----------|------|------|
| 0x7A~0x89 | SLOP, GAM1~GAM15 | 감마 보정 커브. 어두운/밝은 영역의 밝기 분포 조정 |
| 0x4F~0x54 | MTX1~MTX6 | 색상 변환 매트릭스. RGB 색 균형 조정 |
| 0x58 | MTX_SIGN | 매트릭스 부호 비트 |
| 0x24~0x26 | AEW, AEB, VPT | 자동 노출 상/하한 및 동작 영역 |
| 0x9F~0xAA | HAECC1~HAECC7 | 히스토그램 기반 AEC/AGC 제어 |
| 0x6F | AWBCTR0 | 화이트밸런스 제어 |

### 왜 COM7을 여러 번 쓰는가?

`.mem`에서 COM7(0x12)은 4번 등장한다:

| 엔트리 | 값 | 이유 |
|--------|------|------|
| #0 | 0x80 | SW 리셋 (이 값은 자동으로 0으로 돌아감) |
| #2 | 0x00 | defaults 기본값 (VGA, YUV) |
| #43 | 0x11 | QVGA 모드 활성화 (bit4=1) |
| #57 | 0x14 | 최종값: QVGA(bit4) + RGB(bit2) |

SCCB는 write-only이므로, **마지막에 쓰는 값(0x14)이 최종 설정**이 된다.
이전 단계에서 쓴 값은 덮어씌워지지만, 각 단계의 레지스터 세트가 해당 COM7 값을 기대하고 있으므로 순서를 지켜야 한다.
